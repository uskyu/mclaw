# 2026-03-20 - Daily Notes

## PocketBase 后端评估

### 核心发现
- **数据库限制**: PocketBase 仅支持 SQLite (WAL 模式)，官方明确表示不计划支持 MySQL/PostgreSQL
- **商业级使用**: 官方警告尚未达到 v1.0，不建议用于关键生产应用
- **迁移难度**: MVP 阶段可用，但迁移到 PostgreSQL 需要写脚本，用户密码可能需要重置

### 用户关注点
- 用户对 SQLite 稳定性有顾虑（之前遇到过数据库损坏）
- 需要商业级可靠的后端方案
- 偏好 PostgreSQL/MySQL 而非 SQLite
- 希望自托管，不要像 Supabase 那么重

## Appwrite vs Directus 对比

### Appwrite
- **优势**: 开箱即用、BSD开源无限制、功能完整（Auth/Database/Storage/Functions/Realtime）、部署简单
- **劣势**: 只能用 MariaDB、资源占用较高（多容器）
- **适用**: 想要 Firebase 体验，不在乎数据库选择

### Directus
- **优势**: 支持多种数据库（PostgreSQL/MySQL/SQLite/OracleDB）、可视化后台极强、单容器轻量
- **劣势**: BSL 1.1 协议（年收入>$500万需商业许可）、需要自备数据库
- **适用**: 有 PostgreSQL 偏好，需要强大管理后台

**推荐**: 如果追求开箱即用选 Appwrite；如果坚持 PostgreSQL 选 Directus
